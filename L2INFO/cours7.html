DAO : Data Access Object

A data access object (DAO) is an object that provides an abstract interface to some type of database or other persistence mechanism.

DAO : why use it ?

  • Hide the interaction mecanism
  • Centralize all data-acces in same layer

DAO : how to implement ?

  • Create a class representing the data
  • Create a classDAO
  • Implement all interaction with the persistance layer in the DAO

DAO : example


class UserDAO {
    function getAll() {..}
    function save(User $user) {..}
    function delete(User $user) {..}
    ...
}

Singleton

the singleton pattern is a design pattern that restricts the instantiation of a class to one object.

Singleton : why ?

Some ressources should not be loaded more than once because they can be shared across the application. This is mostly for cost or simplicity related reasons.

Singleton : how ?

  • Create private constructor
  • Create private static variable : instance
  • Create public static method : getInstance()
  • getInstance returns content of instance or calls constructor and return instance

Singleton : example


class DB {
    private static $instance;
    private function __construct() {..}
    static public function getInstance() {
        if (self::$instance === null) {
            self::$instance = new DB();
        }

        return self::$instance;
    }
}

Controller

The controller should manage all interaction between the application and the client.

Controller : in a web context

Take all information from the request,
Perform some actions,
Return a response

Controller : in a web context

In a web based context, that means : checking credentials, checking the routes, performing actions based on the route, parse the form and execute the related actions, etc. Eventually, it should return a response.
